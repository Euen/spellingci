Introduction

Spelling CI is a service created by Inaka that will integrate with your Github repositories and check the spelling of your Pull Requests.

Usage

Once you've signed up with your Github user, you'll see a table with a row per repo. To [de]activate Spelling CI for a repo, just select its checkbox button.

When Spelling CI is activated for a repo, a webhook will be added to that particular repository on Github. Then, each time somebody submits a pull request there, Spelling CI will check spelling of the PR's code and report any warnings as github comments.

Configuration

By default Spelling CI will check all the *.md and *.markdown files in your repository, but that can be changed using a spellingci.json file with this format:

{
  "extensions" : ["txt", "other"],
  "ignore_words" : ["word1", "word2"],
  "ignore_patterns" : ["regex1", "regex2"],
  "ignore_blocks" : [{"open" : "open_regex", "close" : "close_regex"}]
}

You can find more info about this configuration file here.

Scope

Spelling CI will check public and private repositories hosted on Github. Spelling CI's code is open-source and it is using many open-source libraries like:

Sheldon
Spelling checker library
Erlang Github
Used to integrate with Github
Cowboy
As the web server that provides both the site and the API to integrate with Github
Rebar3
To be able to compile rebar based projects
Sumo DB
Used for persistency (Gadget keeps track of subscriptions) with a Mnesia backend
Sumo REST
Used to create the REST API
